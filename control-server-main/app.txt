4 roles:
- user: nao tem menus, so tem os botoes para iniciar o voo, ex: Rota1, Rota2. Eses botoes enviam um request para o servidor e ficam a espera que a missao acabe,
- engeneer: pode criar novas rotas, com nome, nome do ficheiro de ida e nome do ficheiro de volta (opcional), e pode fazer upload de ficheiros para o servidor e alterar as coordenadas da base. Tambem pode iniciar o voo (e a aplicacao como temos neste momento). Pelo menos este da para usar no computador (pode ser no browser). Se nao der instalamos uma maquina virtual.
- guest: nao tem menus, so tem um botao que aparece quando o drone esta parado naquela posicao para confirmar que ja recolheu a encomenda.
- admin: gere os utilizadores

requests:

user:
POST request para /run_mission com {'plan_file_name': 'rota_de_ida.plan', (optional)'plan_back': 'rota_de_volta.plan'}

guest:
GET request para /check_return_status em loop. Vai receber uma variavel. Se a variavel for 0 continua a fazer o request, quando a variavel for 1 aparece o botao.
Quando se carregar no botao e realizado este um POST request para /confirm_return (sem json)
